ℹ️ Manual

🌐 Sobre a versão online

Este dashboard vive em tendencia-infra.vercel.app e agora é compartilhado por toda a equipe. Todas as edições (classificações de aditivos, movimentações, aditivos manuais) são sincronizadas em tempo real com um banco na nuvem (Supabase).

Badges no header:

  • 🔗 Conectando… — SDK inicializando (primeiros 1-2 segundos após abrir)
  • ☁️ Sincronizado — tudo em dia com a nuvem ✅
  • ⚠️ Erro ao sincronizar — falha na última operação (passe o mouse pra ver detalhes)
  • 🔴 Offline (local) — sem conexão; alterações ficam só no navegador até voltar

🔐 Login e permissões

Modelo ver = livre / editar = restrito:

  • Sem login — qualquer pessoa com o link consegue ver tudo (badge 👁️ Visualização). Botões de edição ficam escondidos ou cinzas.
  • Logado + email na whitelist — badge fica verde com chip EDITOR. Todos os botões de edição liberam.
  • Logado mas fora da whitelist — badge fica azul com chip SEM PERMISSÃO. Consegue ver, mas não editar. Fale com o admin para ser adicionado.

Como logar: clique em 🔑 Entrar no header, escolha sua conta Google, pronto. Não precisa cadastro nem senha.

📧 Para pedir permissão de edição: envie seu email do Google (o mesmo que usou pra logar) para o admin. Ele adiciona você na whitelist em 10 segundos.

📤 Como atualizar os dados

Todas as atualizações ficam na aba 📤 Uploads (só editores). Dois modos disponíveis:

📊 Upload Completo (Excel) — recomendado:

  • Envie 1 arquivo .xlsx ou .xlsm contendo as 3 abas (Tendência, FlowsValor, Gestões)
  • O dashboard identifica as abas automaticamente pelos nomes
  • Se algum nome for diferente, aparece um modal pra você mapear manualmente
  • As 3 fontes de dados são atualizadas de uma vez só

⚙️ Modo avançado (CSV individual — uso legado):

  • Clique no toggle "Modo avançado" pra expandir os 3 cards separados
  • Envie cada CSV independentemente (útil se atualizou só uma parte dos dados)

Em ambos os modos, cada bloco mostra: 📁 nome do arquivo, 📅 quando/por quem foi enviado, 📤 botão pra substituir e 📜 botão de histórico (últimos 12 arquivos, com opção de baixar de novo).

Nas outras abas, um cabeçalho compacto no topo mostra qual arquivo está alimentando aquela aba (útil pra saber se os dados estão desatualizados).

📦 Backup automático: desde a v0.53, cada arquivo enviado é armazenado no Supabase Storage. O dashboard mantém automaticamente os últimos 12 arquivos por tipo (~1 ano de histórico mensal). Uploads via Excel ficam agrupados pelo mesmo upload_group_id (identificados no histórico com badge via Excel).

🔄 Fluxo mensal recomendado

  1. No começo de cada mês, exporte os 3 CSVs da planilha (TENDÊNCIA, FlowsValor, Gestões)
  2. Faça login no dashboard
  3. Vá em 📤 Uploads e substitua os 3 arquivos (não precisa ser tudo de uma vez)
  4. Confira nos cabeçalhos das outras abas se todas estão apontando pra data certa
  5. Reveja as novas classificações de aditivos que apareceram na aba 🔗 Flows

Os dados classificados anteriormente (aditivos manuais, movimentações, refletidos) continuam preservados mesmo depois de substituir os CSVs.

📈 Aba TENDÊNCIA

Como exportar: Abra a planilha .xlsm, vá na aba TENDÊNCIA, Arquivo → Salvar Como → CSV UTF-8 (.csv).

Estrutura esperada (v0.55+, 22 colunas):

  • A — Chave (ex: 42-21O-1-31005-S05765-ADM5189-01.01.01.02)
  • B — Código hierárquico (ex: 01.01.01.02)
  • C — Código de serviço (ex: S05765) · -1 quando vazio
  • D — Código de insumo (ex: ADM5189) ← obrigatório para o item ser "folha". -1 ou INSUMOS = grupo
  • E — Nome do item
  • F — (vazia — separador visual)
  • G — ORÇ. LICITAÇÃO (R$)
  • H — Corrigido IPCA (R$) ← agora vem antes do INCC
  • I — Corrigido INCC (R$)
  • J — Gestão atual (R$) — cabeçalho contém a data da gestão (ex: GESTÃO 05-2026)
  • K — Diferença (R$) — calculada na planilha
  • L — (vazia — separador visual)
  • M — 🆕 EVOLUÇÃO TEÓRICA (Prevision) — % em cronograma físico
  • N — 🆕 EVOLUÇÃO FINANCEIRA — % gasto do orçamento
  • O — (vazia — separador visual)
  • P — TENDÊNCIA (manual, ainda não usada no dashboard)
  • Q — DIF. TENDÊNCIA (manual, ainda não usada)
  • R — (vazia — separador visual)
  • S — Observações (manuais)

💡 Linha 2 é o subheader com totais globais (área vendável, valores totais e % globais de evolução). A linha 1 é o cabeçalho de nomes, dados começam na linha 3.

Linhas: linha 1 = cabeçalho. A partir da linha 2 são os dados.

Cabeçalho esperado contém (v0.55+): ORÇ. LICITAÇÃO + IPCA + INCC + EVOLUÇÃO

✨ Novidade v0.55: o arquivo agora tem 22 colunas (era 17). Foram adicionadas EVOLUÇÃO TEÓRICA e EVOLUÇÃO FINANCEIRA (integração com o Prevision) para calcular a Aderência Físico-Financeira da obra. Custos indiretos não entram nessa conta.

⚠️ Formato anterior (17 colunas, sem EVOLUÇÃO) não é mais aceito a partir da v0.55.

🔗 Aba FlowsValor

Como exportar: Vá na aba FlowsValor da planilha, Salvar Como → CSV UTF-8.

Estrutura esperada (21 colunas):

  • A — Nº da alteração (numérico)
  • B — Nº ADT
  • C — Departamento (Finalizado / Projeto / Cancelado / Planejamento / Orçamento / Obra)
  • D — Descrição da alteração
  • E — Data de criação (dd/mm/aaaa ou número serial)
  • F–I — Aprovador (depto/nome), Solicitante (depto/nome)
  • J — Custo Total Flowmaster (R$)
  • K — Custo Total Planejamento (R$)
  • L — Motivo
  • M — Justificativa
  • N–Q — Inclusões em Orçamento / Planej. / Tendência / Revisão
  • R — INSUMO PLANEJAMENTO (destino do aditivo) ← essencial
  • S — INSUMO DE REMANEJAMENTO (origem do aditivo, se aplicável) ← essencial
  • T — Observações

Linhas: as 2 primeiras são cabeçalho (linha 1 = título, linha 2 = headers). Dados a partir da linha 3.

Cabeçalho esperado contém: INSUMO PLANEJAMENTO ou CONTROLE DE ALTERAÇÕES

Importante: as edições e os aditivos manuais que você cria neste dashboard NÃO são apagados quando você sobe um novo CSV. Eles ficam preservados.

📅 Aba Gestões

Como exportar: Aba Gestões da planilha → Salvar Como → CSV UTF-8.

Estrutura esperada (27 colunas):

  • B — Empreendimento (filtro: Jardins Zurique)
  • D — Nome da gestão (ex: GESTÃO 05-2026, Atual)
  • H — Classificação financeira (filtro: Obra)
  • K — Key_planejamento (chave igual à coluna A da Tendência)
  • L — Val_totalliquido (R$ por mês de pagamento)
  • J — Mês de pagamento
  • Z — Insumo (ex: I001148, ADM5189)

Lógica: o dashboard soma Val_totalliquido por Key_planejamento e Gestão, filtrando apenas Jardins Zurique e Obra e chave contendo -21O-.

Cabeçalho esperado contém: Descr_gestao ou Key_planejamento

💡 Dicas gerais

  • Formato dos CSVs: UTF-8 com separador ; (padrão BR do Excel).
  • Valores em R$ podem vir com . de milhar e , de decimal — o dashboard converte automaticamente.
  • Datas podem ser dd/mm/aaaa ou número serial Excel (ex: 46147) — converte automaticamente.
  • O dashboard detecta automaticamente o tipo do CSV pelo cabeçalho e rejeita se você subir o errado.
  • Tudo roda offline no navegador — nada é enviado para servidores.
  • Aditivos manuais e classificações ficam salvos no localStorage do navegador. Para mover para outra máquina, exporte o CSV de alterações (botão na aba Flows).
